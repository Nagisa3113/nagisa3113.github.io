解决方案

  1. 编码 Code
  2. 规划 Plan

编码(Hard Code)

每个逻辑节点对应一个确定的动作(可选: 模糊逻辑)

你需要：向机器解释如何实现其目标, 完全控制但工作量大

  • Finite State Machine
  • Behavior

Unity Asset: Play Maker, Behavior Designer

规划(Plan)

建立问题的数学模型，让机器求解

你需要： 制定行为间的关系

  • A* 寻路
  • GOAP

Unity(2019.2) Package: AI Planner (preview)

学习(Learn)

制定游戏规则，让AI在大量训练中找到最佳解

  • 强化学习
  • 模仿学习

你需要：

(强化学习)设置观察量和奖励，让机器在不断试错中找到最佳方法

(模仿学习)作出正确的选择让AI进行模仿

Unity Package: ML-Agents Toolkit(v2.0 beta)

ML-Agents

使用PPO算法，通过TensorFlow进行训练，使用Python API与Unity环境进行交互

Academy

全局环境，控制场景中训练的对象和学习策略

Brain

用来观测并进行决策，每个GameObject上可以有多个Brain控制不同的行为决策，多个GameObect可以使用同一个Brain进行自我博弈

Agent

相当于一个AI物体，需要添加Brain，可以有自身的观察环境（第一人称相机），需要定义在训练中进行的行为

一般情况下我们只需要定义以下参数

  1. 观测向量(Vector Observation Space)

    从环境中观察到的信息,向量存储于键-值对(key-value)中,

    • 连续(Continuous) eg: transform, rotation
    • 离散(Discrete) eg: jump action

  2. 行为向量(Vector Action Space)

    经过计算预计得到的向量，这里不需要定义向量的具体含义，并在之后定义根据该向量采取的具体动作,

    • 连续(Continuous)
    • 离散(Discrete)

  3. 定义行为对应的奖励

    float类型 一般 -1.0~+1.0

  4. 训练参数 eg: max step, learning rate, time scale, layers

    修改config.yaml

训练

使用jupyter进行交互

jupyter notebook

使用trainer_config中的参数进行训练

mlagents-learn <config-path> --env=<env-name> --run-id=<run-identifier> --train

使用tensorboard进行观测

tensorboard --logdir=summaries

ML-Agents Toolkit提供的更多训练方式

  • 使用硬编码或玩家控制器下的角色与AI控制下的角色进行对比训练
  • Visual Observation也就是观察空间摄像机(pixel)
  • Soft Actor-Critic有记忆的学习
  • Curriculum Learning阶段性学习，逐次增加训练难度
  • Imitation Learning模仿学习，AI根据玩家行为进行学习
  • Monitor监视器，随时在UI上显示环境参数
  • Unity Editor, Excutable, Command Line
